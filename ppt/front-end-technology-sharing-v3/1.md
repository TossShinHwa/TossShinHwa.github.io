#### 技术选型
  - JS Framework <!-- .element: class="fragment highlight-blue" -->
  - CSS Framework
  - JS 替代品
  - CSS 替代品


- Why JS Framework  - 如果什么框架也没用，要么不是在做应用，要么out了。  - 从某种意义上讲，无论你选择哪个，都能得到同样的好处。  - 选择会深刻影响你的架构。


- 共识  - MV*  - 数据绑定  - 许可和源代码控制


- 分歧  - 库，插到既有架构中，补充特定功能。  - 框架，只要你遵守，那剩下的就全都是处理通用需求了。


- 优缺点一览
  - Backbone
  - Knockout  - AngularJS  - Polymer
  - React


- Backbone  - \+ 广大社群支持者，教程完善  - \+ 没有改变HTML写法  - \- 基本需求  - \- 相较于其他框架，会面临如何把强大的功能"降级"


- Knockout  - \+ 学习成本低  - \+ 兼容IE6  - \- 语法不优雅  - \- 性能问题


- AngularJS
  - \+ Model-View-Whatever  - \+ 复用 directive  - \+ 优雅 template  - \+ 简易 routing  - \- 学习成本高  - \- 创造过多的概念
